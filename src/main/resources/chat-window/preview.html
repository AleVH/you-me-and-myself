How do I implement a singleton in Kotlin?
14:22

In Kotlin, the idiomatic way to create a singleton is with the object declaration:

KOTLIN 
object DatabaseConnection {
    val url = "jdbc:postgresql://localhost/mydb"

    fun connect() {
        println("Connecting to $url")
    }
}

The object keyword creates a thread-safe singleton that is lazily initialized on first access. You use it directly:

KOTLIN 
DatabaseConnection.connect()

If you need a singleton within a class (companion singleton), use companion object:

KOTLIN 
class MyClass {
    companion object {
        const val MAX_RETRIES = 3
        fun create(): MyClass = MyClass()
    }
}

Key differences from Java:

  • No private constructor boilerplate
  • Thread-safe by default (JVM guarantees)
  • Can implement interfaces and extend classes

What about testing singletons?
14:25

Singletons are notoriously hard to test because they carry state across tests. Two practical approaches:

1. Interface extraction — have the singleton implement an interface, then inject a mock in tests:

KOTLIN 
interface DbConnection {
    fun connect()
}

object RealDb : DbConnection {
    override fun connect() { /* ... */ }
}

2. Reset method — add a @VisibleForTesting reset function (use sparingly).
